# &#x09;			**IAM (Identity and Access Management)**











* ##### used to authenticate users and authorize their access to resources securely.



##### **Key Components of IAM :-**





##### **1. Identification :**



###### &#x09;	User claims identity (username, ID)





##### 2\. Authentication :



###### &#x09;		Verifies identity

###### 

###### &#x09;		**Methods:**

###### 

###### &#x09;			1. Password

###### 

###### &#x09;			2. OTP

###### 

###### &#x09;			3. Biometrics





##### **3. Authorization :**



###### &#x09;		Determines what a user can access

###### 

###### &#x09;		Based on roles/permissions





##### **4. Accounting (Auditing) :**



###### &#x09;			Tracks user activity

###### 

###### &#x09;			Logs actions for security





##### **Examples :**



###### &#x09;	Logging into email with password + OTP

###### 

###### &#x09;	Employee accessing only their department data

###### 

###### &#x09;	Admin having full system access





##### **Real-world Tools :** Active Directory, Okta, AWS IAM







#### **Key Points (Interview) :**



* ###### IAM = Authentication + Authorization

###### 

* ###### Follows least privilege principle

###### 

* ###### Uses multi-factor authentication (MFA)

###### 

* ###### Centralized access control
